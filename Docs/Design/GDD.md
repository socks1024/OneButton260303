# One Button 260303

## 题材限制

- 使用 Godot 开发
- 第一人称 3D 视角，采用 3D 场景与 2D 角色（Billboard Sprite）的美术方案
- 游戏为一键式游戏，只能通过空格键的按住/松开操作接受输入

---

## 游戏概念

玩家被困在一个噩梦中，目标为逃出噩梦。玩家会自动奔跑，在噩梦的道路上坚持跑完全程即可通关。

核心体验：**在恐惧中抉择——睁眼面对恐惧，还是闭眼逃避现实。**

---

## 操作方式

| 操作 | 效果 |
|------|------|
| 松开空格（默认） | 睁眼状态：可以看到前方道路和鬼怪，看到鬼怪时恐惧值持续上升，未看到鬼怪时恐惧值持续下降 |
| 按住空格 | 闭眼状态：屏幕变黑，看不到任何东西，恐惧值不会上升，但迷失值持续上升 |

---

## 核心玩法

### 自动奔跑

- 玩家角色自动向前奔跑，玩家无法控制移动
- 游戏总路程固定，跑完全程即为通关
- 暂定通关时长约 **5 分钟**

### 睁眼状态（默认）

- 玩家拥有正常的第一人称视野，可以看到前方的道路
- 如果视野中出现了鬼怪，恐惧值会持续上升
- 如果视野中没有鬼怪，恐惧值会持续下降（与闭眼时相同）
- 遇到断路时，角色会自动跳过断口，安全通过

### 闭眼状态（按住空格）

- 屏幕变黑/加上闭眼遮罩效果，玩家失去所有视觉信息
- 不会看到鬼怪，恐惧值持续下降
- 迷失值持续上升
- 遇到断路时，由于看不到路况，角色不会跳跃，直接掉入断口 → **游戏失败**

---

## 威胁系统：鬼怪

### 出现机制

- 鬼怪以 2D Sprite（Billboard）形式出现在 3D 场景中
- 在玩家视野范围内的道路前方/两侧随机位置生成
- 每只鬼怪存在一段时间后自动消失

### 感知提示

- 鬼怪出现时伴随音效提示（如低语、脚步声、尖叫等）
- **音效停止 = 鬼怪消失**，这是闭眼状态下判断是否安全睁眼的唯一线索
- 不同种类的鬼怪可以有不同的音效特征（后续可扩展）

### 生成规则

- 鬼怪在玩家前方/两侧的随机位置生成，存在一段时间后自动消失，再过一段时间又会出现
- 随玩家奔跑距离推进：
  - 鬼怪出现频率增加
  - 单次出现的鬼怪数量增加
  - 鬼怪存在时间变长
  - **鬼怪生成位置逐渐靠近玩家**（初期较远，后期贴近）
  - **玩家的恐惧值增长速率逐渐上升**（鬼怪变得更恐怖）

---

## 地形系统：断路

### 基本规则

- 道路上会随机生成断口（缺口/深坑）
- 断口在玩家视野范围内可以提前看到
- **睁眼状态**遇到断口 → 角色自动跳过，安全通过
- **闭眼状态**遇到断口 → 角色掉入 → **游戏失败（即死）**

### 生成规则

- 断口之间保持一定最小间距，避免连续断口无法应对
- 断口不会与鬼怪出现时间完全重叠（给玩家留下决策窗口）
  - 但后期可能出现短暂重叠，增加压力
- 随游戏进程推进：
  - 断口出现频率增加
  - 由于移动速度加快而视野不变，玩家看到断口后的反应时间缩短

## 数值系统

### 恐惧值

| 属性 | 说明 |
|------|------|
| 最大值 | 100 |
| 初始值 | 0 |
| 增长方式 | 睁眼状态下看到鬼怪时持续增长 |
| 下降方式 | **未看到鬼时持续下降**（闭眼、或睁眼但视野内无鬼） |
| 满值后果 | 游戏失败（被吓死） |

- 基础增长速率：**X 点/秒**（视野中有鬼怪时）
- 随玩家奔跑距离推进，增长速率逐渐提高（鬼怪变得更恐怖）
- 多只鬼怪同时出现时，增长速率叠加
- 未看到鬼时以 **Rf 点/秒** 的速率下降恐惧值（闭眼或睁眼无鬼=心态恢复）

### 迷失值

| 属性 | 说明 |
|------|------|
| 最大值 | 100 |
| 初始值 | 0 |
| 增长方式 | 闭眼状态下持续增长 |
| 下降方式 | **睁眼时持续下降** |
| 满值后果 | 游戏失败（永远迷失在梦境中） |

- 基础增长速率：**Y 点/秒**（闭眼时）
- 迷失值是对玩家闭眼时间的总量限制，迫使玩家不能一直闭眼逃避
- 睁眼时以 **Rl 点/秒** 的速率下降迷失值（睁眼=找到方向）

### 移动速度

| 属性 | 说明 |
|------|------|
| 初始速度 | V₀ |
| 增长方式 | 随存活时间线性/曲线增长 |
| 影响 | 视野不变，速度越快对断路的反应窗口越短 |

---

## 难度曲线

游戏时长约 5 分钟，难度随时间推进逐步提升：

| 阶段 | 时间段 | 特征 |
|------|--------|------|
| 新手期 | 0:00 - 1:00 | 鬼怪少且存在时间短，断口稀疏，速度慢。让玩家熟悉机制 |
| 成长期 | 1:00 - 2:30 | 鬼怪频率和持续时间增加，断口开始增多，速度逐渐提升 |
| 压力期 | 2:30 - 4:00 | 鬼怪频繁出现，扣 San 速率明显提高，断口与鬼怪开始出现短暂重叠窗口 |
| 冲刺期 | 4:00 - 5:00 | 高频鬼怪+高速移动+频繁断口，玩家需要精确判断闭眼/睁眼时机 |

---

## 胜负条件

### 胜利条件

- 在恐惧值和迷失值均未达到满值的情况下，跑完全程到达终点

### 失败条件（满足任一即失败）

| 失败条件 | 触发方式 |
|----------|----------|
| 恐惧值满 | 睁眼状态下看到太多鬼怪，被吓死 |
| 迷失值满 | 闭眼时间过长，永远迷失在梦境中 |
| 掉入断口 | 闭眼状态下经过断路 |

## UI 界面

### 游戏中 HUD

- **恐惧值条**：显示当前恐惧值，可用渐变色表示危险程度（绿 → 黄 → 红）
- **迷失值条**：显示当前迷失值，可用渐变色表示危险程度（绿 → 黄 → 红）
- **进度指示**：显示当前跑了多远 / 距离终点还有多远（进度条形式）
- **闭眼遮罩**：闭眼时覆盖屏幕的黑色遮罩（可带呼吸/心跳动效）

### 游戏结束画面

- 胜利：逃出噩梦的画面 + 通关信息
- 失败：根据不同失败原因显示不同画面/文字
  - 恐惧值满："你被恐惧吞噬了……"
  - 迷失值满："你永远迷失在了梦境之中……"
  - 掉入断口："你在黑暗中坠入了深渊……"

## 音频设计

### 环境音

- 持续的噩梦氛围背景音（低频嗡鸣、风声、远处的回响等）
- 随难度提升，环境音逐渐变得更压抑

### 鬼怪音效

- 每只鬼怪出现时播放对应音效
- 音效持续时间 = 鬼怪存在时间
- **音效停止是玩家判断安全的关键信号**
- 音效具有空间感（3D 音效），可辨别鬼怪方位

### 状态音效

- 闭眼时：心跳声（随迷失值升高心跳加速）
- 恐惧值高时：耳鸣/杂音干扰
- 接近断口时：可考虑风声变化等细微提示（增加听觉玩法深度）
