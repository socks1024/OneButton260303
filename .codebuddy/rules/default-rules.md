## 开发要求

### 功能模块开发

增加新功能或新逻辑前，先看文档和项目中现有的内容，能用预先做好的东西，就不要重复造轮子。

项目模板及其中功能的说明在 res://Docs/Template/README.md 这个文档中，文档开头有目录，可以根据目录查询相关功能。

### 数据与配置
- `@export` 变量数值有问题时，**只在聊天中告知，不要自行修改代码或 .tscn**
- 能在场景中静态配置的内容，**不要在脚本中动态生成**
- 可调节的变量**优先 `@export` 导出**，不要写成脚本内不可调常量

## 编码规范要求

编写代码后，必须进行语法检查，不符合语法的部分必须修正（比如缩进不符合规范）。

### GDScript
- 注释使用**中文**
- 禁止 `:=` 自动推断，变量和函数返回值**必须显式标注类型**
  - ✅ `var count: int = 0` / `func get_name() -> String:`
  - ❌ `var count := 0` / `func get_name():`

### GDShader
- **禁止 `return;`**，用条件分支代替提前返回

### 工具偏好
- 日志：优先用 **clog** 插件，不用 `print` / `push_warning` / `push_error`
- 相机：优先用 **Phantom Camera** 插件，不用原生 `Camera2D` / `Camera3D`

## BUG 处理要求

修改有时会因为引擎的问题而失败，如果用户反馈了“没变化”之类的bug，需要优先检查上一次的更改是否被正确保留了。
